# تحدي ضياء — كأس العالم 2026

موقع تنبؤات لكأس العالم 2026 مع حساب نقاط **تلقائي بالكامل**.
السيرفر يجيب النتائج الحقيقية من الإنترنت كل 30 دقيقة، ويحسب نقاط كل
مشارك تلقائياً — بدون أي تدخل بشري بعد الإطلاق.

## كيف يشتغل

- **السيرفر** (`server.js`) يجيب بيانات كأس العالم من مصدر مجاني ومفتوح
  (openfootball/worldcup.json) كل 30 دقيقة، ويستخرج نتائج المباريات.
- **كل مستخدم** عنده `userId` فريد يتخزن بمتصفحه، وتوقعاته تتخزن على
  السيرفر.
- **حساب النقاط** يصير على السيرفر تلقائياً: 2 نقطة للتوقع الصحيح،
  1 نقطة لو الفريق الصحيح تأهل "بديلاً" عن فريق تانٍ كان متوقع.
- **صفحة الترتيب** (Leaderboard) تعرض كل المشاركين مرتبين حسب نقاطهم —
  تلقائياً، أي حدا يدخل عليها يشوف الترتيب الحالي.

## النشر على Render.com (مجاني، يشتغل 24/7)

### 1. ارفع المشروع على GitHub
أنشئ repository جديد وارفع هذا المجلد كامل.

### 2. أنشئ Web Service على Render
1. روح [render.com](https://render.com) وسجّل دخول (تقدر تستخدم GitHub
   مباشرة)
2. اضغط **New** → **Web Service**
3. اربطه بالـ repository
4. الإعدادات:
   - **Name:** `tahaddi-diaa-2026` (أو أي اسم)
   - **Build Command:** `npm install`
   - **Start Command:** `node server.js`
   - **Instance Type:** Free
5. اضغط **Create Web Service**

### 3. خلّيه يبقى شغال (مهم!)
الخطة المجانية على Render بتنام بعد 15 دقيقة بدون زيارات. عشان يضل
شغال 24/7 بدون ما تفتح شي:
- استخدم خدمة مجانية متل [UptimeRobot](https://uptimerobot.com) أو
  [cron-job.org](https://cron-job.org) عشان "تزور" رابط السيرفر كل 10
  دقائق (مثلاً `https://your-app.onrender.com/api/health`)
- هيك السيرفر يضل صاحي دايماً، وحساب النقاط يستمر تلقائياً.

### 4. شارك الرابط
بعد ما يخلص الـ Deploy، Render بيعطيك رابط مثل:
```
https://tahaddi-diaa-2026.onrender.com
```
هذا الرابط هو **كل التطبيق** — افتحه من أي جهاز وأي شبكة. شاركه مع
رفقاتك مباشرة.

## الملفات

```
wc2026-server/
├── server.js          ← السيرفر (Node.js + Express)
├── package.json
├── public/
│   ├── index.html      ← واجهة المستخدم
│   └── app.js           ← منطق الواجهة
└── data/
    └── predictions.json ← تخزين توقعات كل المشاركين (يُنشأ تلقائياً)
```

## ملاحظات تقنية

- مصدر النتائج: `openfootball/worldcup.json` على GitHub — بيانات عامة
  مجانية بدون أي API key، يُحدّثها فريق مستقل يومياً تقريباً.
- لو حبيت تستخدم مصدر أسرع بالتحديث، بدّل قيمة `SOURCE_URL` داخل
  `server.js` لمصدر آخر بنفس الصيغة.
- بيانات كل المشاركين تتخزن بملف JSON بسيط على السيرفر (`data/predictions.json`).
  لو الموقع كبر كتير ممكن لاحقاً ننقلها لقاعدة بيانات حقيقية.
